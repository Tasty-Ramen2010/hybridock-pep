### TestReceptorPrep (covers PREP-01)

Fixture: a DockConfig pointing to tests/fixtures/receptor_tiny.pdb and a tmp_path output_dir.
DockConfig construction requires the receptor_path to exist, so use the fixture file directly.

Tests:
1. `test_prepare_receptor_calls_prepare_receptor4` — monkeypatch subprocess.run to return
   CompletedProcess(returncode=0); assert the call happened with "prepare_receptor4.py" in args[0].
   Also assert pdbqt_path was returned and output_dir / "receptor.pdbqt" exists.

2. `test_prepare_receptor_nonzero_exit_raises_prep_error` — monkeypatch subprocess.run to return
   CompletedProcess(returncode=1, stderr="fatal: bad input"); assert pytest.raises(PrepError)
   and that the error message contains "fatal: bad input" (captured stderr in message).

3. `test_prepare_receptor_always_regenerates` — call prepare_receptor twice with the same config
   (monkeypatched subprocess). Assert no exception; just proves no caching guard crashes the second call.

4. `test_pdbfixer_called_before_subprocess` — use a call recorder: monkeypatch pdbfixer to record
   that findMissingResidues/findMissingAtoms/addMissingHydrogens were called. Use a mock PDBFixer
   class that records method calls. Assert all three methods called before subprocess.run is invoked.
   Implementation hint: monkeypatch `hybridock_pep.prep.receptor.PDBFixer` with a class that sets
   instance flags when each method is called; monkeypatch `hybridock_pep.prep.receptor.PDBFile.writeFile`
   to write a dummy PDB; monkeypatch subprocess.run to return success.

For pdbfixer mock, since PDBFixer and PDBFile.writeFile are external, mock at the module import
level in receptor.py. The simplest approach: monkeypatch `hybridock_pep.prep.receptor.PDBFixer`
with a spy class.
