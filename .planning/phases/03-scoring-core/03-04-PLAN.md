The script reads a 3-column training CSV (D-08: pdb_id, peptide_sequence, experimental_pkd)
and a separate --scores-json file containing pre-computed vina_score and ad4_score values.

Why this design: fit_calibration() needs vina_scores, ad4_scores, n_residues_list, and
experimental_pkd. The training CSV provides peptide_sequence (for n_residues) and
experimental_pkd. The --scores-json provides vina_score and ad4_score per complex (keyed by
pdb_id). Running live Vina scoring inside calibrate_alpha.py is out of scope for Phase 3
(Phase 5 CLI will wire calibrate to the full pipeline). If --scores-json is not provided,
the script aborts with a clear message.

Implement `parse_args(argv=None)` function returning an argparse.Namespace:
- `--training-csv` (required): Path to 3-column D-08 input CSV (default: data/training_complexes.csv)
- `--scores-json` (optional, dest="scores_json"): Path to JSON mapping pdb_id → {vina_score, ad4_score}. If not provided, script aborts with a clear error.
- `--output` (required): Path to write calibration.json (default: data/calibration.json)
- `--verbose` / `-v`: Enable DEBUG logging

Implement `main(args=None)`:
1. Parse args if not provided: `args = parse_args(args)`
2. Configure logging (INFO by default, DEBUG if --verbose)
3. Validate --scores-json is provided:
   ```python
   if args.scores_json is None:
       raise ValueError(
           "--scores-json is required for Phase 3 calibration; "
           "run hybridock-pep calibrate in Phase 5 for live scoring"
       )
   ```
4. Load scores JSON: `scores = json.loads(Path(args.scores_json).read_text())`
   Each entry: scores[pdb_id] = {"vina_score": float, "ad4_score": float}
5. Read training CSV with csv.DictReader; collect lists:
   - vina_scores: [float(scores[row["pdb_id"]]["vina_score"]) for row in rows]
   - ad4_scores: [float(scores[row["pdb_id"]]["ad4_score"]) for row in rows]
   - n_residues_list: [len(row["peptide_sequence"]) for row in rows]  # derived from CSV, not scores JSON
   - pkd_list: [float(row["experimental_pkd"]) for row in rows]
   If a pdb_id from the CSV is missing from scores JSON: raise KeyError with clear message.
6. Log INFO: `"Calibrating on %d complexes from %s", n_complexes, training_csv`
7. Import from hybridock_pep.scoring.entropy:

10. Log INFO: `"Calibrated: α=%.3f β=%.3f r=%.3f RMSE=%.3f → %s", result["alpha"], result["beta"], result["pearson_r"], result["rmse_kcal_mol"], args.output`
11. Validate with load_calibration: `from hybridock_pep.scoring.entropy import load_calibration; load_calibration(Path(args.output))`
    This is a post-write self-check — if α/β converged outside bounds the script aborts here
    with the informative ValueError rather than silently writing a corrupt calibration.

End with:

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| data/training_complexes.csv → calibrate_alpha.py | CSV rows parsed with csv.DictReader; float() conversion can raise ValueError on corrupt data |
| --scores-json → calibrate_alpha.py | JSON file parsed; KeyError raised if pdb_id missing; float() conversion validates score values |
| fit_calibration output → write_calibration → load_calibration | Post-write self-check: load_calibration validates the just-written file; catches any optimizer boundary convergence issue |
| scripts/ → src/ | Script imports from hybridock_pep package; package must be installed (pip install -e .) in score-env |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Tampering | training_complexes.csv injection (corrupt float in experimental_pkd) | mitigate | float() conversion in CSV parsing raises ValueError with row context; script aborts before calling fit_calibration |
| T-03-14 | Tampering | scores-json injection (corrupt float in vina_score/ad4_score) | mitigate | float() conversion on scores JSON values raises ValueError; script aborts before calling fit_calibration |
| T-03-15 | Tampering | calibration.json post-write tampered before load_calibration check | accept | Self-check is belt-and-suspenders; primary protection is the validation in load_calibration at dock-time |
| T-03-16 | Denial of Service | optimizer convergence at boundary (α=0.2 or α=1.2) | accept | L-BFGS-B bounds handle this correctly; load_calibration post-check would only fail if bounds exceeded, which L-BFGS-B prevents by design |
| T-03-17 | Information Disclosure | Pearson r and RMSE in calibration.json | accept | Diagnostic statistics; no PII or model secrets; intentionally public metadata |
