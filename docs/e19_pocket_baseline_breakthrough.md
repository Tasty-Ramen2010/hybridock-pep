# E19 — Pocket-baseline recovery: real WITHIN crystal-65, does NOT generalize cross-source

**Date:** 2026-06-10. After E18 v2 returned NO, tested three attacks on the per-protein-
baseline wall: H1 pocket-baseline recovery, H2 learned favorability, H3 ligand-efficiency.
**H2/H3 = NO. H1 looked like a breakthrough (crystal-65 LOO r=0.576) but step-2 validation
(`e19_validate.py`) DEMOTED it: it does NOT survive the leave-dataset-out acid test.**

> **CORRECTION (step-2):** An earlier draft claimed "pocket hydrophobicity (poc_eis) is the
> dominant axis, |r|≈0.73." That was a **degenerate-prediction artifact** — single-feature
> LOO produced near-constant predictions (std 0.089) so the correlation was meaningless.
> Raw corr(poc_eis, ΔG) on crystal-65 = **−0.03**. The claim is RETRACTED.

## Validation verdict (step 2): crystal-65-specific, not a general law

- **Leave-DATASET-out (acid test):** train pocket+interface on crystal-65 → predict PEPBI
  distinct-family means = **−0.28** (and reverse −0.07). The model trained on one source
  ANTI-predicts the other. **Does not transfer across data sources.**
- **Sign universality FAILS:** features that predict in crystal-65 flip in PEPBI
  (poc_n +0.41/−0.11; poc_f_hyd −0.42/+0.23; sasa_hb +0.47/−0.29). No universal physics.
- **Pooled 82-distinct-target LOO = +0.40–0.45** (ridge-stable) but this is partly a
  two-cluster artifact (LOO leaks each dataset's mean), NOT honest per-target prediction.
- **Confound:** PEPBI structures are largely Rosetta-MODELED (memory: ~10 kcal/mol bias),
  so part of the failure may be PEPBI pocket-feature NOISE rather than pure physics failure.
  Cannot disentangle without a second CLEAN-crystal distinct-target set.

**The crystal-65 r=0.576 is real WITHIN that curated set (permutation-validated) but is
dataset-specific CALIBRATION, not a generalizable predictor. NOT promoted. NOT wired.**

## Step-2b — DATASET PROVENANCE + POSE-DISTRIBUTION audit (Ram's flag, confirmed)

**Bias 1 — PEPBI is 100% Rosetta theoretical models.** Every one of 320 PEPBI structures
carries `EXPDTA THEORETICAL MODEL / MODEL GENERATED BY ROSETTA`. So the leave-dataset-out
test (crystal→PEPBI = −0.28) was real-crystal-geometry → Rosetta-model-geometry: a
GEOMETRY-SOURCE mismatch, not a clean physics-transfer test. The step-2 "does not
generalize" verdict is CONTAMINATED by model-quality differences and should not be read as
proof the physics fails. (It also does not prove it succeeds — we simply can't tell from
these two sources.)

**Bias 2 — everything was scored on CRYSTAL ORACLE poses.** crystal-65, the 276-complex
calibration set, and all E18/E19 features use the crystal peptide position. `dataset_analysis.md`
§8 already states crystal-pose scores are "an upper bound on what HybriDock-Pep can achieve."
The deployed tool scores RAPiDock DIFFUSION poses (noisier). We have ZERO affinity-labeled
RAPiDock poses (gen_n100 poses have no Kd; Kd sets have only crystal/model poses).

**Pose-robustness curve (simulated scatter, `e19_pose_robustness.py`, no GPU):** displaced
crystal peptides to target RMSD, recomputed features, re-ran crystal-65 LOO:

| pose RMSD | pocket | interface | pocket+interface |
|---|---|---|---|
| 0.0 (oracle) | 0.546 | 0.365 | 0.576 |
| 1.0 | 0.521 | 0.278 | 0.473 |
| 2.0 (~typical RAPiDock) | 0.509 | 0.182 | 0.409 |
| 3.0 | 0.450 | 0.218 | 0.384 |

**Finding: the two terms degrade OPPOSITELY.** The POCKET term is pose-robust (0.51 @ 2 Å —
it reads receptor residues lining the site, stable under peptide displacement). The INTERFACE
term (H-bond/aromatic — the r≈0.45 within-target ranker we planned to ship) COLLAPSES (0.18 @
2 Å — contacts are pose-fragile). Realistic deployed absolute-ΔG ≈ 0.40, carried by the
pose-robust pocket term.

**Implication (revises the plan):**
- The pose-robust POCKET term is the MORE deployable signal — it tolerates RAPiDock pose
  error and needs only an approximate binding site. Its weakness is cross-source calibration
  (Bias 1), not pose noise.
- The within-target INTERFACE ranker takes a real haircut on RAPiDock poses (0.45 oracle →
  ~0.18–0.28 realistic). Its advertised number is an upper bound.
- CAVEAT: simulated rigid+jitter scatter ≠ real RAPiDock error (which is STRUCTURED — right
  backbone ladder, wrong rotamer/register). Real poses could preserve interface features
  better or worse. This is a proxy, not the real measurement.

**The one decisive experiment left (needs GPU):** generate RAPiDock N=100 on the 65 Kd-labeled
crystal complexes, recompute pocket+interface on top-cluster-centroid poses, re-run the honest
harness. That is the only way to get the true deployed number. GPU not visible in current shell.

## The result

Crystal-65, leave-one-out, **absolute cross-target ΔG** (the regime that was −0.5 / sign-flipped):

| model | LOO Pearson r | RMSE (kcal/mol) |
|---|---|---|
| interface only (H-bond/aromatic/BSA — old approach) | +0.365 | 2.02 |
| **pocket descriptors only** | **+0.546** | 1.80 |
| **pocket + interface** | **+0.576** | 1.78 |
| mean-baseline | — | 2.13 |

- **Permutation-validated:** pocket-only r=+0.51 vs 200× shuffled-label null (95th pct +0.24,
  max +0.39). Real, not 8-features-over-65-points overfit.
- **Dominant axis = pocket hydrophobicity** (`poc_eis` single-feature |r|≈0.73; `poc_f_hyd`
  +0.37, `poc_n` size +0.36). Physics: deep hydrophobic pockets bind peptides tighter,
  largely independent of which peptide. This IS the per-target baseline α_j.

**r=0.576 clears the CLAUDE.md §8 target (Pearson r ≥ 0.55 on held-out).**

## Why this works where everything else failed

ΔG_abs = α_j (per-target baseline) + Σβ·x_ij (peptide interface) + ε. Every prior experiment
tried to predict ΔG from the *interface* term alone → size-confounded, sign-flipped (−0.54).
**The baseline α_j was the missing term, and it is predictable from the receptor pocket** —
just not from the peptide interface. Pocket hydrophobicity/size recovers α_j at r≈0.55.

## Scope limits (harshest-critic — where it does NOT hold)

1. **Needs structurally-DISTINCT targets.** Works on crystal-65 (65 different receptors,
   α-spread 2.13). FAILS on PEPBI (r=−0.29): PEPBI "families" are sequence variants on
   near-identical pockets → no pocket variance to exploit. So pocket-baseline helps when
   comparing *different proteins*, not variants of one.
2. **Weights do NOT cross-dataset transfer.** crystal→PEPBI with pocket features = −0.40.
   The r=0.576 is leave-one-out *within* crystal-65 (in-distribution cross-target). The
   linear calibration is dataset-specific; only the SIGN/physics (hydrophobic→tighter) is
   expected universal. Production use = calibrate on a diverse multi-target panel, apply
   within a comparable distribution. Re-calibration per deployment, not plug-and-play.
3. **Pocket is pose-defined** (receptor residues within 8 Å of the docked peptide). Valid
   for a docking tool (you have a pose) but the "pocket" is where the peptide sits — mild
   self-reference, physically legitimate (it is the binding site).
4. **Predicts target bindability, barely uses the peptide.** Pocket-only r=0.55 ≈ "this
   pocket binds peptides tightly in general." Useless for ranking DIFFERENT peptides vs ONE
   fixed target (pocket constant there) — that still needs the interface term (+0.45).
5. **crystal-65 selection bias:** crystallized complexes skew toward ordered, tight,
   well-defined pockets. Real-world floppy/shallow sites may be predicted worse.

## The clean three-regime story (what to actually ship)

| use case | regime | feature | accuracy |
|---|---|---|---|
| Rank DIFFERENT targets / absolute ΔG across distinct receptors | cross-target | **pocket + interface** | **r≈0.55–0.58** (NEW) |
| Rank DIFFERENT peptides vs ONE target | within-target | interface H-bond/aromatic | r≈0.45 |
| **Selectivity ΔΔG (peptide vs two distinct receptors — the iGEM PfLDH/hLDH case)** | both | pocket gives baseline OFFSET, interface gives specificity | strongest combined |

**Selectivity is the sweet spot:** PfLDH and hLDH are *distinct pockets*, so the
pocket-baseline term (its strongest regime) supplies the ΔΔG offset, and the interface term
supplies peptide specificity. The thing the iGEM tool exists to do is exactly where the new
signal is most useful.

## E26 — pose-quality audit (Ram: "are we scoring only good poses?")

**NO — we score MEDIOCRE poses, and pose quality does NOT drive affinity accuracy.**
RMSE/r are 1 prediction PER COMPLEX (65), not per pose; the pose strategy only picks WHICH
pose feeds the features. What we actually scored:
  - rank1 (pose_0, deployment default): median 3.4Å RMSD, only 9% <2Å, 35% >4Å
  - RAPiDock MIS-RANKS: best-of-100 exists (median 1.6Å, 68% <2Å) but is buried below rank1.
geometry+MJ LOO by pose choice (kcal/mol):
  - rank1     r=0.564 RMSE 1.80
  - top5_mean r=0.496 RMSE 1.91
  - bestrmsd (oracle pose selection, median 1.6Å) r=0.467 RMSE 1.96  ← NOT better!
corr(|prediction error|, rank1 pose RMSD) = −0.05 ≈ 0.

Implications: (1) we are NOT cherry-picking good poses — the real-pose numbers use median
3.4Å rank1, so they are pessimistic, not inflated. (2) Our pocket+interface+MJ features are
POSE-ROBUST: they read the binding site + approximate placement, so a 3.4Å pose scores about
as well as a 1.6Å pose. Better pose RANKING is headroom for pose-RMSD, NOT for affinity.

## E31 — WHY features flipped, and the FIX (Ram: physics is universal, encoding was wrong)

Confirmed: the sign-flips were a SIZE/Simpson confound in our EXTENSIVE encodings, NOT the
physics. Recomputed INTENSIVE + desolvation-aware features on crystal-65 + the-98:

| feature | crystal-65 | the-98 | sign-consistent? |
|---|---|---|---|
| mj_per_contact (avg, NOT sum) | +0.295 | +0.206 | YES — vs mj_contact (sum) flipped −0.47/+0.32 |
| f_hyd_iface (hydrophobic FRACTION) | −0.422 | −0.247 | YES (strongest single) |
| frac_pol_satisfied (desolv-aware H-bond) | +0.190 | +0.150 | YES — vs hb_count flipped +0.31/−0.13 |
| bsa_hyd | −0.281 | −0.312 | YES |

Mechanism: extensive features (hb_count, sasa_sb, mj_contact-sum, arom_cc, poc_n) secretly encode
INTERFACE SIZE. Within a narrow set size~affinity; across diverse targets it does not → sign flips.
Hydrophobic burial is the exception (extensive quantity IS the physics, ~const/Å²). For polar/
electrostatic terms the physics is NET energy after desolvation, not the raw count → flips.

FIX = intensive encoding: mj_PER_CONTACT (composition quality), f_hyd_iface (hydrophobic
fraction), frac_pol_satisfied (satisfied-H-bond fraction = desolvation-aware). These do NOT flip.

Generalization (pooled 163 LOO): bsa_hyd 0.370 | f_hyd_iface 0.412 | universal-intensive-4 0.421.
The properly-encoded multivariate set BEATS single-feature on adequate data (0.421>0.370) — so
"one signal carrying everything" was an artifact of covariance-overfitting on 65 points, not a law.
Old extensive-11 transfer was −0.14; new intensive set generalizes. Full resolution: intensive
encoding + more diverse data (harvest in progress) = the path. f_hyd_iface is the new best single
universal feature. scripts/e31_intensive_features.py; /tmp/e31_intensive.json.
